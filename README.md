# Immune Profiling Dashboard

A Python and SQLite pipeline for analyzing peripheral blood immune cell composition across two CPI-7 checkpoint inhibitor trials, paired with a Streamlit dashboard for interactive exploration.

---

## Background

Checkpoint inhibitors produce durable responses in some patients but not others, and predicting who will respond before or early in treatment remains an open clinical problem. One hypothesis is that pre-existing immune cell composition in peripheral blood — particularly cytotoxic T cell levels — distinguishes eventual responders from non-responders.

This project ingests longitudinal PBMC immune profiling data from two independent cohorts, computes sample-level population frequencies, and runs a stratified statistical analysis comparing responders to non-responders across five immune cell populations at multiple timepoints.

---

## Dataset

Two parallel cohorts, both treated with CPI-7 and sampled by PBMC at days 0, 7, and 14 from treatment start:

| Cohort | Project | Subjects | Responder rate |
|--------|---------|----------|----------------|
| Melanoma | PROJ-MEL-01 | 40 | 40% |
| NSCLC | PROJ-NSC-01 | 40 | 40% |

Five immune populations are measured per sample: B cells, CD8 T cells, CD4 T cells, NK cells, and monocytes.

**Note on synthetic data:** `data/cell-count.csv` is generated by `generate_data.py` (seed 42). One biologically motivated signal is embedded at baseline and the pipeline is designed to recover it. All other populations and post-baseline timepoints are noise. The synthetic design allows the statistical methods to be validated against a known ground truth rather than an unknown one.

## Setup and Usage

### Install dependencies

```
make setup
```

### (Optional) Regenerate the synthetic dataset

```
make generate
```

The committed `data/cell-count.csv` is already the output of this command with seed 42. Only re-run if you want to produce a fresh dataset.

### Run the full analytical pipeline

```
make pipeline
```

This will:
1. Drop and rebuild `immune_trial.db` from `data/cell-count.csv`
2. Compute per-sample immune population frequencies
3. Run responder vs non-responder analysis, pooled and stratified by day (0, 7, 14)
4. Write baseline cohort summaries and longitudinal trajectory aggregates
5. Save all outputs to `outputs/`

### Launch the dashboard

```
make dashboard
```

Opens at http://localhost:8501 by default (check terminal output for the exact URL).

## Statistical Methods

Responder vs non-responder comparisons are run independently for each immune cell population across four analyses: pooled across all timepoints and separately for each of days 0, 7, and 14.

### Test

A **two-sided Mann–Whitney U test** compares relative frequency distributions between response groups per population. The non-parametric test is appropriate because immune cell percentages are not assumed to be normally distributed.

### Multiple Testing Correction

**Benjamini–Hochberg (BH) FDR correction** is applied within each analysis across the five population-level p-values. Correction is scoped per analysis — p-values from different timepoints are not combined.

### Repeated Measures

Each patient contributes up to three PBMC samples (days 0, 7, 14). The day-specific analyses are straightforward — one sample per subject at a given timepoint is one independent observation. The pooled analysis is more subtle: treating all three timepoints as independent would artificially multiply subjects and inflate statistical power. To avoid this, each subject is collapsed to a single per-population mean across their available timepoints before the MWU test is run.

| Analysis | Observation unit | Independence |
|----------|-----------------|--------------|
| Day 0 / Day 7 / Day 14 | One sample per subject | Independent by construction |
| Pooled (all) | Per-subject mean across available timepoints | One value per subject per population |

### Significance Threshold

Statistical significance is defined as q < 0.05 after FDR correction. Nominal p-values are reported alongside q-values for transparency, but conclusions are based on FDR-adjusted q-values.

## Reproducibility

The project rebuilds deterministically from scratch:

- `make setup` installs dependencies
- `make generate` regenerates `data/cell-count.csv` (seed 42)
- `make pipeline` drops and rebuilds the database and regenerates all outputs

No manual steps are required between setup and a fully populated dashboard.
